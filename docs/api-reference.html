üìö API Reference

Complete REST API documentation for the Threat Modeling Platform

üåê Base URL

https://localhost/api

All API requests should be made to this base URL. The API accepts JSON payloads and returns JSON responses.

üîê Authentication

Login

Authenticate and receive a JWT token for subsequent requests. 

POST /api/auth/login

Request Body

{
  "username": "admin",
  "password": "secure-password"
}

Response (200 OK)

{
  "token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 1,
    "username": "admin",
    "role": "admin",
    "email": "admin@example.com"
  },
  "expiresIn": "24h"
}

cURL Example

curl -X POST https://localhost/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"secure-password"}' \
  -k

Using the Token

Include the JWT token in the Authorization header for authenticated requests: 

Authorization: Bearer eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...
Note: Tokens are signed using RS256 (RSA asymmetric encryption) and expire after 24 hours.

üóÇÔ∏è Threat Models

List Threat Models

GET /api/threat-models

Query Parameters

  • page - Page number (default: 1)
  • limit - Results per page (default: 20)
  • status - Filter by status (draft, in-review, approved)
  • search - Search by name or description

Response (200 OK)

{
  "data": [
    {
      "id": 1,
      "name": "E-Commerce Platform",
      "description": "Cloud-based e-commerce system",
      "created_at": "2025-01-15T10:30:00Z",
      "updated_at": "2025-01-15T14:20:00Z",
      "status": "approved",
      "threat_count": 12,
      "high_severity_count": 3
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 45,
    "pages": 3
  }
}

cURL Example

curl -X GET "https://localhost/api/threat-models?page=1&limit=10" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -k

Update Threat Model

PUT /api/threat-models/:id

Request Body

{
  "status": "approved",
  "components": ["web-server", "api-gateway", "database", "storage", "cdn"]
}

Response (200 OK)

{
  "message": "Threat model updated successfully",
  "updated_fields": ["status", "components", "updated_at"]
}

Delete Threat Model

DELETE /api/threat-models/:id

Response (204 No Content)

Returns empty body on successful deletion.

‚è±Ô∏è Rate Limiting

The API implements rate limiting to ensure fair usage. Limits are applied per IP address and authenticated user.

Default Limits

  • Anonymous: 60 requests/hour
  • Authenticated: 1000 requests/hour
  • Analysis endpoints: 10 requests/hour

Rate Limit Headers

X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 847
X-RateLimit-Reset: 1642687200
Note: When rate limit is exceeded, the API returns a 429 status code with a Retry-After header.

üíª SDK Examples

JavaScript (Node.js)

const axios = require('axios');

// Login
const { data } = await axios.post(
  'https://localhost/api/auth/login',
  { username: 'admin', password: 'pass' }
);

// Use token
const threats = await axios.get(
  'https://localhost/api/threat-models',
  {
    headers: {
      'Authorization': `Bearer ${data.token}`
    }
  }
);

Python

import requests

# Login
response = requests.post(
    'https://localhost/api/auth/login',
    json={
        'username': 'admin',
        'password': 'pass'
    },
    verify=False
)
token = response.json()['token']

# Use token
threats = requests.get(
    'https://localhost/api/threat-models',
    headers={'Authorization': f'Bearer {token}'},
    verify=False
).json()

cURL

# Login and save token
TOKEN=$(curl -X POST \
  https://localhost/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"pass"}' \
  -k | jq -r '.token')

# Use token
curl -X GET \
  https://localhost/api/threat-models \
  -H "Authorization: Bearer $TOKEN" \
  -k
